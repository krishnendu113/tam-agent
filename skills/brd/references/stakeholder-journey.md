# Stakeholder Journey — PM → Client → Sign-Off

This file maps the journey of human stakeholders across the pipeline: who they are, when they are involved, what they must do, and what sign-offs are required. Updated in v7.1 to reflect the BRD Evaluation Layer and Clarification Register.

---

## Stakeholder Map

| Role | Organisation | Involved In | Authority |
|------|-------------|------------|----------|
| **PM (Project Manager)** | Capillary | All phases | Drives delivery; owns Key Gaps Report; escalates risks |
| **Solutions Lead / BA** | Capillary | Phase 01 (BRD validation + evaluation review) | Validates technical feasibility; reviews evaluation scores |
| **Tech Lead** | Capillary | Phase 01–03 (integration sections) | Confirms API and integration specs; owns G5/G7 guardrail items |
| **Client BPO** | Client | Phase 01 (BRD approval) + Phase 02 (discovery) | Must sign off BRD for Gate 1; approves waivers in Clarification Register |
| **Client Project Sponsor** | Client | Phase 01–02 (oversight) | Approves scope, budget, go-live; final authority on Critical Gap waivers |
| **Client Tech Lead / IT** | Client | Phase 01–03 (integration) | Confirms POS, API, infra readiness; owns G3/G6 guardrail items |
| **Client Marketing Owner** | Client | Phase 01–02 (programme design) | Defines tiers, earn rules, communication preferences |
| **Capillary CSM** | Capillary | Post Gate 2 | Manages commercial and support SLA |

## Phase 01 — Stakeholder Journey (BRD Creation + Evaluation)

```
Capillary PM
    │
    ├─ Schedules kick-off meeting with client stakeholders
    ├─ Runs Question Sets A–F (agent-guided)
    ├─ BRD V1.0 generated by agent
    │
    ├─ STEP 5 EVALUATION AUTO-RUNS
    │    → Key Gaps Report produced
    │    → Gap counts: Critical / Risk / Warning
    │    → PM reviews all Critical Gaps
    │
    ├─ PM sends draft BRD + Key Gaps Report to Solutions Lead for internal review
    │
    └─ Solutions Lead reviews + validates
         │
         └─ PM incorporates feedback → resolves Critical Gaps → BRD V1.1
              │
              └─ Sends BRD + updated Key Gaps Report to Client BPO for review
                   │
                   ├─ Client BPO raises queries / change requests
                   │    └─ PM updates BRD (V1.2…) → re-runs evaluation → resubmit
                   │
                   └─ Client BPO satisfied — all Critical Gaps resolved
                        │
                        └─ Client BPO APPROVES → signs Document Approval table
                             │
                             └─ ✅ Gate 1 CLEARED
```

**Sign-off required at Gate 1:**
- Client BPO signature on Document Approval table (or formal email confirmation)
- Key Gaps Report saved to Confluence with zero open Critical Gaps
- Capillary PM and Solutions Lead sign-off on Step 6 Quality Checklist
- Signed BRD saved as `[ClientName]_BRD_V[X.X]_Signed.pdf` to Confluence › BRD Documents
- Key Gaps Report saved as `[ClientName]_Key_Gaps_Report_V[X.X].txt` to Confluence › BRD Documents

## Phase 02 — Stakeholder Journey (Discovery)

```
Capillary PM + Agent
    │
    ├─ Imports open Risk Items + Warnings from Key Gaps Report into Guardrail Status Panel
    ├─ Builds Discovery Dashboard (prefilled from signed BRD)
    ├─ Schedules discovery session(s) with client
    │
    └─ Discovery Session(s)
         │
         ├─ Client Marketing Owner confirms: programme rules, tiers, communication preferences
         ├─ Client Tech Lead confirms: POS readiness, API ownership, infra, credential availability
         ├─ Client Project Sponsor confirms: budget, phasing, hard deadlines, go-live commitment
         │
         ├─ PM tracks completion score in real time
         ├─ PM logs every resolved gap in the Clarification Register
         │
         └─ All Must-Have items answered + Yes/No section confirmed?
              │
              ├─ Guardrail gaps outstanding?
              │    └─ PM assigns owners + due dates
              │         ├─ RESOLVE → update BRD if needed → re-run Step 5 → update register
              │         └─ WAIVE → named approver (Client BPO or Sponsor) provides written approval
              │
              └─ ✅ Gate 2 CLEARED
                   → PM exports Guardrail Scorecard + Discovery .txt + Clarification Register
```

**Sign-off required at Gate 2:**
- Discovery export `.txt` reviewed by client BPO or project sponsor
- Guardrail Scorecard showing all 7 categories status (exported and saved)
- Clarification Register showing all gaps Resolved or Waived with named approver
- BRD updated to latest version (vX.X) incorporating all discovery findings
- All waivers have written approval on record — verbal agreement not sufficient

## Phase 03 — Stakeholder Journey (Jira)

```
Capillary PM + Agent
    │
    ├─ Creates all Epics and Stories in Jira (PSV project)
    ├─ Verifies every Story has a BRD FR reference
    ├─ Flags any UNCHARTED stories → confirms with PM
    │
    └─ PM reviews all tickets with Tech Lead
         │
         ├─ Confirms story points, priorities, and sprint assignments
         ├─ Client Tech Lead reviews integration-related stories
         ├─ Any UNCHARTED stories confirmed as in-scope or Change Request
         │
         └─ Exports CSV + XLSX → saves to Confluence › Jira Backlog
              │
              └─ ✅ Phase 03 Complete — Pipeline Done
                   → Stakeholders notified of live Jira board
```

## Change Request (CR) Process

For any scope change after BRD sign-off (Gate 1):

```
1. Requestor submits CR to Capillary PM
2. PM assesses impact: scope, timeline, cost — within 5 business days
3. Impact assessment shared with client
4. Client approves / rejects CR in writing
5. Approved CR:
   → Appended to BRD as new major version (V2.0)
   → Step 5 evaluation re-run on updated BRD
   → Key Gaps Report updated
   → New Jira Stories created for approved CR scope
```

Approval path for CRs above agreed threshold:  
Client BPO → Capillary PM → Capillary Management (per SOW terms)

## Escalation Path

```
Issue identified
    │
    ▼
PM → Client BPO / Capillary Solutions Lead
    │ (unresolved after 5 business days)
    ▼
Client Project Sponsor / Capillary Account Manager
    │ (unresolved after 10 business days)
    ▼
Steering Committee
```

## Communication Cadence (standard recommendations)

| Meeting | Frequency | Attendees | Purpose |
|---------|-----------|----------|---------|
| Weekly Status Call | Weekly | PM, Client BPO, relevant workstream leads | Progress, blockers, next steps |
| Discovery Session | As needed (usually 2–4 sessions) | PM, Client Marketing + Tech stakeholders | Close open discovery items |
| Evaluation Review | Once per BRD version | PM, Solutions Lead, Client BPO | Walk through Key Gaps Report findings |
| Steering Committee | Monthly or at gate milestones | Project Sponsor, PM, Senior Capillary Lead | Strategic decisions, budget, scope |
| UAT Sign-Off Session | Once per environment | PM, QA lead, Client BPO, Client Tech | UAT completion and go/no-go |
